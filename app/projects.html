glimpse

Backend Development

Glimpse is a shared calendar application that makes planning group events easier and organized. This application uses React, an Express Server, Node, and MongoDB to store the users shared calendars.

For this application I was a member of the back-end team. I worked to create routes between the client-side, server-side, and the database. I worked on user authentication using the Node.JS package, Passport. We wanted to make sure that a user had their own unique log-in information and could add other users to the groups that they created by using that user’s email address. I also worked with React and React Router to render different pages and re-render when the user updated their information.

The database for this application has a many to many relationship, meaning the user can be associated with many groups and the groups and can associated with many users. The groups are then broken down into different events. The use of Mongo database worked best for this application due to the flexibility of what the data can look like with the database.

My Medical Diary

Backend Development

My Medical Diary is a handy application for storing notes from medical visits, appointments, to do items, and doctors' information. For this application we used a mySQL database, Sequalize and Handlebars Node packages, Javascript, and Bootstrap 4.

While making My Medical Diary, I worked on the back-end elements of the application. I worked with a MySQL database and used the Node.JS package Sequalize to create routes between the front-end, Express Server, and the database. I also used the Node.JS package Handlebars to have the user’s database information render in the correct place on the page. This application’s database has a one to many relationship. In this case the user can have many notes, to-do items, appointments, and doctors.

News Scraper

News Scraper

Full Stack Development

News Scraper is an application that scrapes articles from the Smithsonian Magazine a website and displays them on the screen. The user is then able to save those articles in a database and write notes about those saved articles. This applications uses HTML, CSS, Javascript, MongoDB, and Node packages Express and Mongoose.

In this application I used a Cheerio news scraper to pull headlines and the URL from the Smithsonian Magazine website and then show on the page. If the user decides they like an article they can click the button to save that article to the database. I created routes that save that article to the MongoDB and then render that information on the Saved Articles Page. There is a similar interaction occurring when the user writes a note about the article.

For the design of this application I used Bootstrap3 and tried to keep the interface simple and user-friendly. The database used has a one to many association, where the saved article can have many notes associated with it.
